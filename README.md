### Ответы на контрольные вопросы

1.  **Какие типы данных поддерживает SQLite? Как в SQLite можно хранить логические значения и даты?**

    SQLite поддерживает пять основных типов хранения (storage classes): `NULL` (отсутствие значения), `INTEGER` (целое число), `REAL` (число с плавающей точкой), `TEXT` (строка в кодировке UTF-8/UTF-16) и `BLOB` (двоичные данные, хранящиеся «как есть»). В отличие от многих других СУБД, SQLite использует динамическую типизацию — тип привязан к конкретному значению, а не к столбцу.

    Отдельных типов `BOOLEAN` и `DATE`/`DATETIME` в SQLite нет. Логические значения принято хранить как `INTEGER`: 0 — `false`, 1 — `true`. Даты и время можно хранить тремя способами: как `TEXT` в формате ISO-8601 (`"2025-04-01 12:30:00"`), как `INTEGER` в виде Unix-timestamp (количество секунд с 1 января 1970 г.) или как `REAL` (юлианский день). Для работы с датами SQLite предоставляет встроенные функции `date()`, `time()`, `datetime()`, `strftime()` и другие.

2.  **Для чего нужен класс `SQLiteOpenHelper`? Опишите назначение методов `onCreate()` и `onUpgrade()`.**

    `SQLiteOpenHelper` — это абстрактный вспомогательный класс из Android SDK, который упрощает управление жизненным циклом базы данных: её создание, открытие и обновление версий. Он инкапсулирует логику проверки существования файла БД и её текущей версии, избавляя разработчика от ручного управления этими процессами.

    Метод `onCreate(SQLiteDatabase db)` вызывается **один раз**, когда база данных создаётся впервые (файл БД ещё не существует). В нём размещают SQL-команды для создания таблиц и начального наполнения данными (`CREATE TABLE`, `INSERT`).

    Метод `onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion)` вызывается, когда версия базы данных, переданная в конструктор хелпера, **выше**, чем версия, записанная в уже существующем файле БД. Он предназначен для миграции схемы: добавления/удаления таблиц и столбцов, преобразования данных и т. д. Параметры `oldVersion` и `newVersion` позволяют организовать пошаговую миграцию через цепочку `if`/`switch`.

3.  **В чем разница между методами `getWritableDatabase()` и `getReadableDatabase()`? В каких ситуациях может возникнуть ошибка при вызове `getWritableDatabase()`?**

    `getWritableDatabase()` открывает (или создаёт) базу данных в режиме чтения и записи. `getReadableDatabase()` пытается сделать то же самое, но если открытие на запись невозможно, он откроет БД в режиме «только чтение» вместо того, чтобы бросить исключение. Таким образом, `getReadableDatabase()` — более «мягкий» вариант.

    Вызов `getWritableDatabase()` может завершиться ошибкой (`SQLiteException`) в следующих ситуациях: на устройстве закончилось свободное место на диске; файл БД повреждён (corruption); файловая система смонтирована в режиме «только чтение» (например, при проблемах с хранилищем); отсутствуют необходимые права доступа к файлу. В подобных случаях рекомендуется использовать `getReadableDatabase()`, если операция записи не требуется.

4.  **Что такое `Cursor`? Как правильно перемещаться по его элементам и почему важно закрывать его после использования?**

    `Cursor` — это интерфейс, представляющий результирующий набор строк, полученных в ответ на SQL-запрос. Он работает по принципу итератора: указатель («курсор») первоначально стоит перед первой строкой, и его можно перемещать по набору.

    Типичный паттерн перемещения:
    ```java
    if (cursor.moveToFirst()) {
        do {
            String name = cursor.getString(cursor.getColumnIndexOrThrow("name"));
            // обработка строки
        } while (cursor.moveToNext());
    }
    ```
    Помимо `moveToFirst()` и `moveToNext()`, доступны методы `moveToLast()`, `moveToPrevious()`, `moveToPosition(int)`, `getCount()` и другие.

    Закрывать `Cursor` (вызывать `cursor.close()`) важно потому, что он удерживает системные ресурсы: соединение с БД, память под буферы строк, файловые дескрипторы. Незакрытые курсоры приводят к утечкам памяти и могут исчерпать лимит открытых курсоров, что в конечном итоге вызовет сбой приложения.

5.  **Что такое `ContentValues` и для каких операций он применяется?**

    `ContentValues` — это класс-контейнер, который хранит набор пар «имя столбца — значение». По сути это обёртка над `HashMap<String, Object>`, специально предназначенная для взаимодействия с базой данных.

    Он применяется в операциях вставки (`insert()`) и обновления (`update()`) записей в таблице. Разработчик добавляет значения методами `put("column_name", value)`, а затем передаёт объект `ContentValues` соответствующему методу `SQLiteDatabase`:
    ```java
    ContentValues values = new ContentValues();
    values.put("name", "Иван");
    values.put("age", 25);
    db.insert("users", null, values);
    ```
    `ContentValues` обеспечивает типобезопасность, автоматическое экранирование значений (защита от SQL-инъекций) и удобный API для формирования данных строки.

6.  **В чем отличие методов `query()` и `rawQuery()`? Приведите пример использования `rawQuery()` с параметром-плейсхолдером (`?`).**

    Метод `query()` — это «конструктор запросов»: он принимает отдельные параметры (имя таблицы, массив столбцов, строку условия `WHERE`, аргументы, `GROUP BY`, `HAVING`, `ORDER BY`, `LIMIT`) и собирает из них SQL-запрос самостоятельно. Это снижает вероятность синтаксических ошибок и делает код более читаемым.

    Метод `rawQuery()` принимает готовую SQL-строку целиком и массив аргументов для плейсхолдеров `?`. Он даёт полный контроль над запросом (можно писать `JOIN`, подзапросы, `UNION` и т. д.), но требует от разработчика самостоятельного формирования корректного SQL.

    Пример `rawQuery()` с плейсхолдером:
    ```java
    String sql = "SELECT name, age FROM users WHERE age > ? AND city = ?";
    String[] args = {"18", "Москва"};
    Cursor cursor = db.rawQuery(sql, args);
    ```
    Плейсхолдеры `?` подставляются по порядку из массива `args`, что защищает от SQL-инъекций и упрощает параметризацию запросов.

7.  **Как обработать ситуацию, когда таблица уже существует, но её структура была изменена (например, добавлено новое поле)?**

    Для этого используется механизм версионирования базы данных через `SQLiteOpenHelper`. Необходимо увеличить номер версии БД в конструкторе хелпера (например, с 1 до 2). При следующем обращении к базе система обнаружит, что текущая версия файла БД ниже запрошенной, и автоматически вызовет метод `onUpgrade()`, в котором и выполняется миграция:

    ```java
    @Override
    public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {
        if (oldVersion < 2) {
            db.execSQL("ALTER TABLE users ADD COLUMN email TEXT DEFAULT ''");
        }
        if (oldVersion < 3) {
            db.execSQL("ALTER TABLE users ADD COLUMN phone TEXT");
        }
    }
    ```
